¿Cómo se usa?

El objecto Required es una interface del objecto Watchers donde se pueden declarar que campos o grupos de formularios sean requeridos para efectuar un submit. Recibe como argumento un objeto de propiedades que definen algunos aspecto de su comportamiento como los callbacks.

Ejemplo del objeto de configuración:

        {
            required: boolean,
            callbacks: {
                validate: function() {},
                reset: function() {}
            }
        }

Realizando una selección con jQuery podemos asignarle una configuración de la siguiente forma:

        $("#Foobar").required();

Además, el componente soporta algunos shorthands que se pueden concatenar con el método and() facilitando su instanciación con otras validaciones de distintos tipos, por ejemplo: 

        $("#Foobar").email().and().required();

Cada vez que se instancia un componente de la familia Watchers éste retorna un objecto público de control que nos brinda información sobre la instancia y nos permite ejecutar algunos métodos públicos.

        var foo = $("#Foobar").required(); // Same example
        
        // returns
        foo = {
            uid: 121,                         //  Global number instance
            element: HTMLInputElement,        //  Input DOM Interface
            type: "watcher",                  //  Component Type
            status: boolean,                  //  Status
            reference: HTMLInputElement,      //  Positioner reference
            parent: HTMLFormElement,          //  Form parent
            validations: {                    //  Validation Map
                required: true                
            },
            messages:{},                      //  Messages Map
            and(),                            //  Concatenate method 
            reset(),                          //  Reset the value
            validate(),                       //  Execute the validations
        }

Configuración

Parámetros de configuración que puede recibir este componente:

text
boolean: Si se define true valida que sea texto válido: a-z y A-Z.
email
boolean: Si se define true valida que sea email válido.
url
boolean: Si se define true valida que sea una url válida.
minLength
number: Se define con un número entero y valida que el valor no sea inferior a ese número.
maxLength
number: Se define con un número entero y valida que el valor no sea superior a ese número.
messages
object: Este objeto permite definir mensajes predefinidos para cada tipo de validación configurada.
callbacks
object:
validate
Recibe una función que se ejecutará como respuesta del componente luego de efectuar las validaciones definidas.
reset
Recibe una función que se ejecutará como respuesta del componente luego de efectuar el reseteo del valor del campo.
